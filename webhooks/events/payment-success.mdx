The `payment.success` event is sent when a payment has been **successfully processed** and completed.  
You can use this webhook to update transaction status, notify users, or trigger fulfillment logic in your system.

---

## Example Payload

```json
{
  "event": "payment.success",
  "id": "99036bd3-a8b2-403c-867c-3d03dafbe4b7",
  "data": {
    "payment_id": "b91a0f36-b10b-4a31-ae12-e8b4c5ee71f4",
    "status": "success",
    "amount": 599
  },
  "eventGeneratedTime": 1756451608.069325
}
````

---

## Data Fields

<ResponseField name="payment_id" type="string">
  Unique identifier for the payment transaction.
</ResponseField>

<ResponseField name="status" type="string">
  Current status of the payment (always `success` for this event).
</ResponseField>

<ResponseField name="amount" type="integer">
  Payment amount in the smallest currency unit (e.g., cents for USD).
</ResponseField>

---

## Handling the Event

When you receive a `payment.success` event:

1. **Verify the signature** to ensure authenticity.
2. **Check the `payment_id`** in your system and mark it as completed.
3. **Acknowledge with a `200 OK`** response within 30 seconds.

```bash
# Example handler (Node.js/Express)
app.post('/webhook', express.raw({ type: 'application/json' }), (req, res) => {
  const event = JSON.parse(req.body);
  if (event.event === 'payment.success') {
    updateOrderStatus(event.data.payment_id, 'completed');
  }
  res.sendStatus(200);
});
```
